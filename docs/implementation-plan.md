# Implementation Plan

## 1. まず決めること

MVP を実装する前に、以下を固定します。

- 対象人数: まずは 1 名または少人数に限定するか
- 対象シーン: 遊び、食事、午睡のどれから始めるか
- 識別方法: 個人識別をするか、まずはエリア単位の観測に留めるか
- 出力形式: 連絡帳本文だけか、時系列イベント一覧も出すか

初手としては、以下の前提が現実的です。

- 1 台のカメラ
- 1 つの保育室
- 個人識別はまだ強くやらない
- まずはエリア滞在や時刻イベントを扱う
- 連絡帳で頻出の `食事` `お昼寝` `遊び` を優先対象にする

## 2. 推奨フェーズ

### Phase 1: 配信基盤の固定

目的:

- カメラ映像を安定受信できる状態を作る

実施内容:

- Raspberry Pi の ffmpeg 送信を常駐化
- MediaMTX の起動方法を固定
- 接続確認手順をスクリプト化

成果物:

- 起動手順
- systemd ユニット
- 動作確認コマンド

### Phase 2: 受信ワーカー

目的:

- RTSP ストリームを Python から読めるようにする

実施内容:

- OpenCV または PyAV で RTSP 読み込み
- 定期サンプリング
- タイムスタンプ付きフレーム保存

成果物:

- `ingest_worker`
- 設定ファイル
- 受信ログ

### Phase 3: イベント抽出

目的:

- 連絡帳に使える最小限のイベントを生成する

実施内容:

- ROI 設定
- 動体 / 滞在ベースのルール作成
- イベントの開始・終了ロジック作成
- `食事` `お昼寝` `遊び` の 3 テーマに対する観測項目を固定

成果物:

- イベント JSON
- イベント保存テーブル

### Phase 4: 下書き生成

目的:

- イベント列から読みやすい文章を作る

実施内容:

- テンプレート生成
- イベント欠損時のフォールバック
- 保育士向けの編集前提 UI
- 頻出テーマごとに、保護者向けの短い定型文パターンを用意

成果物:

- 連絡帳ドラフト
- 生成理由の表示

### Phase 5: 評価

目的:

- 実運用に耐えるかを確認する

実施内容:

- 誤検出の種類を記録
- 保育士の修正量を測定
- 負担軽減効果を確認

成果物:

- 精度評価メモ
- 運用上の制約整理

## 3. 技術選定のたたき台

### 映像配信

- Raspberry Pi
- FFmpeg
- MediaMTX

この部分は既存検証をそのままベースにできます。

### 受信 / 解析

- Python 3.11 以降
- OpenCV
- NumPy
- 必要に応じて PyAV

### API / UI

- FastAPI
- 軽量な HTML テンプレートまたは React

ただし MVP 初期は、API + シンプルなサーバレンダリングで十分です。

### 保存先

- SQLite から開始

保存対象:

- streams
- events
- diary_drafts
- review_logs

## 5. 直近タスク

優先順位は次の通りです。

1. RTSP を読む最小の Python スクリプトを作る
2. フレーム取得結果をログに出す
3. `食事` `お昼寝` `遊び` のいずれか 1 テーマでイベントを 1 つでも出せるようにする
4. そのイベントから、保護者向けの定型文を返す
5. Web 画面で確認できるようにする

## 6. 成功条件

MVP が成立したと言える条件を先に定義します。

- 30 分以上ストリームを安定受信できる
- 少なくとも 1 種類のイベントを自動記録できる
- そのイベントから日報文面の下書きを生成できる
- 人が見て修正可能な形で提示できる

## 7. 今のボトルネック

- 個人識別を入れると難易度が急に上がる
- 映像から直接「気持ち」や「様子」を決め打ちすると誤りやすい
- 生映像保存を前提にするとプライバシー設計が重くなる

したがって、初期段階では次を守るべきです。

- 行動や時刻など観測可能な情報に限定する
- 自動生成文は控えめな表現にする
- 最終判断は保育士が行う前提にする

## 8. ユースケース更新の根拠

連絡帳支援のユースケース整理には、以下の記事を参考にしました。

- 保育のひきだし「もう悩まない！保育士が知りたい『連絡帳を書くコツ』と『使える例文』」
  - 食事・おやつ
  - お昼寝・排泄
  - 遊び
  - 友だちとの関わり / トラブル / 成長
  - 体調の変化や怪我
